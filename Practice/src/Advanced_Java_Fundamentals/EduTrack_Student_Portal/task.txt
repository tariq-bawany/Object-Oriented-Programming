Question 1 — The "EduTrack" Student Portal System
Semester Practical: Advanced Java Fundamentals & Encapsulation
Overview
You have been recruited to build the core engine for EduTrack, a next-generation university student management portal. Your goal is to create a robust StudentProfile class that is bulletproof against invalid data, supports legacy record migration, and handles multi-currency fee payments.

Part 1: The Secure Foundation (Encapsulation)
Create a class StudentProfile with the following constraints:
Private Fields: studentID (String), studentName (String), cgpa (double), enrollmentStatus (String)
Business Logic:

CGPA Integrity: CGPA must always stay between 0.0 and 4.0. It can never go outside this range.
Status Control: enrollmentStatus cannot have a public setter. It can only be changed via enroll() or suspend() methods.
Safe Grade Update: A method updateCGPA(double newCGPA) must verify that the value is between 0.0 and 4.0 before updating.


Part 2: The "Smart" Identity (String Manipulation)
Your system must automatically clean and validate student data:

Auto-Formatting: When setting studentName, the system must Title Case the name and trim() any leading/trailing spaces.
ID Validation: Every studentID must follow the strict format: STU-XXXX-2026

Use startsWith("STU-"), endsWith("-2026"), and length() to verify. Total length must be exactly 13 characters.
Fallout Logic: If the ID is invalid, set the field to "UNASSIGNED".




Part 3: Flexible Enrollment (Constructor Overloading & Chaining)
Implement three ways to register a student, using constructor chaining this(...) to avoid code duplication:

Basic: Takes only studentName — generates a random ID and sets CGPA to 0.0.
Standard: Takes studentName and initialCGPA.
Data Migrator: Takes a single raw String from a legacy CSV file (e.g., "ali khan, 3.5").

Parse the string and chain it to the main constructor so Title Case logic is applied automatically.




Part 4: Multi-Format Fee Processor (Method Overloading)
Implement the processFeePayment method with three different signatures:

Overload 1 (Standard): processFeePayment(double amount) — records a USD fee payment directly (just print a confirmation with the amount).
Overload 2 (Currency Shift): processFeePayment(double amount, String currencyCode)

If PKR: divide amount by 280 to convert to USD, then confirm.
If EUR: multiply amount by 1.1 to convert to USD, then confirm.


Overload 3 (Encoded Voucher): processFeePayment(String voucherEntry)

Parses a string like "FEE:1500". Use .split(":") to extract the value and process the payment.




Part 5: Academic Audit & Analytics
Finalize the class with reporting and comparison logic:

The Masking Method: Create getProfileSummary() which returns a string with the student's name masked for privacy (e.g., A** K***).

Requirement: Use a loop to replace middle characters of each word with asterisks.


The Final Logic: Create a static method compareStudents(StudentProfile a, StudentProfile b).

Return the student with the higher CGPA, but only if both are "ENROLLED".
If one is suspended, return the enrolled one.
If both are suspended, return null.
